## Versioning notes

For manuscript-linked repositories, record the exact release version used for analysis, figure generation, or public archive submission.

Recommended examples:

- `v1.0.0` for a stable public release
- `v1.0.1` for a minor correction
- `v1.1.0` for a method or workflow update
- `v1.0.0-rc1` for a release candidate

Do not silently overwrite released analysis code. Create a new version when results, dependencies, settings, or outputs change.
