## Maintainer responsibilities

Maintainers are responsible for:

- reviewing pull requests
- checking reproducibility-sensitive changes
- keeping documentation current
- updating the changelog when appropriate
- checking dependency and security alerts
- confirming release tags and DOI/archive links
- coordinating with the PI or project lead before public release

## Manuscript-linked repositories

For repositories linked to manuscripts, posters, theses, reports, or public software releases, maintainers should verify that changes do not silently alter:

- analysis settings
- input data handling
- metric definitions
- output files
- figures or tables
- release tags
- DOI/archive records

## Access review

Repository access should be reviewed periodically, especially when students graduate, collaborators leave the project, or the repository becomes public.
