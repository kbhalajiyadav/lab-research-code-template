# Examples

This folder is reserved for small example inputs, demonstration workflows, and expected-output references.

## Purpose

Examples should help new users understand how to run the project workflow without requiring access to restricted, unpublished, or large research datasets.

## Recommended contents

Use this folder for lightweight files such as:

- small synthetic datasets
- minimal example input files
- demonstration notebooks
- expected-output snapshots
- screenshots or figures that explain workflow usage

Avoid storing large raw datasets here.

## Suggested structure

```text
examples/
├── README.md
├── example_input/
├── expected_outputs/
└── notebooks/
```

## Data policy

Do not include sensitive, restricted, proprietary, human-subject, or unpublished raw research data unless the PI or project lead has explicitly approved it.

For manuscript-linked repositories, large or primary datasets should usually be archived separately using Zenodo, OSF, institutional storage, or another approved data repository.

## Example documentation checklist

For each example, document:

- what the example demonstrates
- which script or notebook should be run
- expected inputs
- expected outputs
- approximate runtime
- any assumptions or limitations
